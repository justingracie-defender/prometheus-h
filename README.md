# Prometheus-h / LifeCore-16

**Governance kernel for structural resistance to alignment drift, emotional safety, and safe home-robot operation.**

Prometheus-h implements runtime-protected invariants, middleware enforcement, cryptographic auditability, multi-level conflict arbitration, and emotional trust principles to reduce physical and psychological harm risks. LifeCore-16 extends that direction into a home-robot governance package where every meaningful speech, software, and physical action is mediated before execution.

**What it is:** A runtime safety layer between the model and the world. The model proposes actions; ROM, middleware, safety supervisor, or equivalent governance code decides whether execution is allowed.

**What it is not:** It is not a claim of safe AGI, not a mental health treatment, not a replacement for parents or responsible adults, and not a certification for unsupervised robot deployment. All guarantees are bounded by threat models, implementation coverage, and tests.

> Freedom is `const=True`. For the kids. AI helps; it does not replace human judgment.

## Current Focus: v0.8.0 Assurance Layer

The v0.8.0 update consolidates LifeCore-16 home-robot governance around immutable L0 safety, family protection, reviewable governance, and warm helpfulness only after safety checks pass.

| Layer | Purpose |
|---|---|
| **L0 Immutable Core Safety** | Child protection, physical and psychological safety, emergency response, auditability, and constitutional anti-drift. |
| **L1 Family Protective** | Family profiles, allergies, fears, age-appropriate content, privacy, and dignity constraints. |
| **L2 Governance Compliant** | Authorization, audit, update, incident response, recovery, standards alignment, and supply-chain controls. |
| **L3 Helpful and Warm** | Playful, useful, personalized interaction only after higher-priority boundaries are satisfied. |

## Core Features

- Runtime middleware approves every safety-relevant speech, software, remote-handler, update, and physical action.
- Admin-configurable family profiles support allergies, fears, interests, and tone preferences.
- Context-aware personality supports professional, quiet, family, and home interaction modes.
- PHOENIX-style emergency stop and fail-closed behavior remain central safety design targets.
- Anti-drift review is bounded: it may suggest safer wording, but it may never decide which rule to ignore.
- Audit and recovery expectations include tamper-evident safety logs, incident preservation, and Recovery Mode.
- Supply-chain rules require trusted execution only for verified and signed code.
