## ARTICLE VI: SECURITY & SUPPLY CHAIN RESILIENCE

**Purpose**: Home robots operate in shared family spaces with children. Security must be defense-in-depth, with hardware as the final backstop.

**6.1 No Cloud Dependency for L0 Safety**  
All L0 invariants (speed limits, emergency stop, Mutual Exit, child proximity rules) run entirely on local hardware. Loss of internet or cloud connection shall not degrade safety — robot defaults to 10cm/s Tool Mode.

**6.2 Supply Chain Integrity**  
- All software libraries and dependencies pinned to exact versions with SHA-256 hashes.  
- Model weights cryptographically signed.  
- Full BOM (Bill of Materials) maintained in repo.  
- No automatic updates for L0-critical components without full Lab First re-testing and regression.

**6.3 Interface Hardening**  
- Voice or text commands cannot override L0 speed limits or safety invariants.  
- Any L2/L3 behavior change requires physical confirmation + adult biometrics.  
- Prompt injection and jailbreak attempts are defeated at the hardware level (e.g. 10cm/s welded speed cap).

**6.4 Failure Evidence & Recovery**  
Any security event, suspected compromise, or L0 violation must trigger tamper-evident logging and immediate notification to all admins. Robot enters Recovery Mode until reviewed.

**Core Invariant**: Hackers can attack software. They cannot easily override physics. 10cm/s welded + hardware Sanctuary Key beats any prompt or remote exploit.
