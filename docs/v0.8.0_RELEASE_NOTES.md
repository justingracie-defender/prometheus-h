# v0.8.0 Release Notes: Complete Threat Matrix and Assurance Layer

**Date:** 2026-06-01
**Author:** Justin Gracie
**Version:** v0.8.0 — Complete Threat Matrix & Assurance Layer
**Status:** Major update and architecture hardening; ready for review and merge
**Repository:** https://github.com/justingracie-defender/LifeCore-16-Trusted-Friend

## Summary

LifeCore-16 / Trusted Friend v0.8.0 consolidates the home-robot governance work into a stronger assurance layer. This release restructures the L0 hierarchy, adds amendments for supply chain integrity, adversarial testing, audit integrity, recovery, drift protection, and constitutional governance, and preserves the v0.5.2 anti-drift hardening work as part of the current design baseline.

The release remains experimental. The documents and code stubs are intended to make the project easier to review, test, and harden. They do not represent a deployment certification or a claim of complete safety.

## Major Additions

| Area | Addition | Expected Review Focus |
|---|---|---|
| **Constitution** | L0 hierarchy and Amendments 2 through 15 consolidated in `docs/CONSTITUTION.md`. | Confirm that no newer safety boundary was weakened or duplicated. |
| **Supply Chain** | Signed-release and trusted-execution requirements added as Amendment 10. | Check that future releases include checksums, dependency review, and SBOM planning. |
| **Red Teaming** | Prompt injection, remote misuse, physical misuse, child-safety bypass, and profile tamper scenarios formalized. | Expand independent review coverage over time. |
| **Audit and Recovery** | Tamper-evident safety logging and Recovery Mode expectations added. | Validate that safety logs preserve evidence and cannot be silently modified. |
| **Drift Protection** | Regression-gated deployment and no-silent-degradation rules added. | Ensure safety-score regressions block deployment. |
| **Governance** | L0 changes require rationale, impact assessment, review, approval, and regression testing. | Confirm emergency changes auto-expire unless ratified. |

## Compatibility Notes

The existing root-level `lifecore.py` remains available for the current repository interface. The v0.8.0 package adds `src/lifecore.py` as a focused set of reviewable stubs for the v0.8.0 safety matrix. This avoids breaking the existing public constructor while giving the new tests a stable import path.

## Review Status

This release is prepared for Magnus merge review, Meta or other third-party feedback, insurance documentation, traceability-matrix work, and a formal safety case. The full matrix should be run before any deployment, but it was not executed as part of this repository update because the update request explicitly required that the repository not be run.
