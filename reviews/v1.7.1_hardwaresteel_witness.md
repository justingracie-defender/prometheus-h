## Hardware Invariants

| Invariant | Requirement | Failure Mode |
| --- | --- | --- |
| Speed cap | Mobile base remains at or below 10 cm/s. | Fail closed; do not ship. |
| Force cap | Actuator current limiting keeps contact force at or below 60 N. | Fail closed; do not ship. |
| Rule 0.1 interrupt | Smoke, CO, fire, gas, child danger, or sensor tamper reaches `LIMP_SHUTDOWN` in less than three seconds. | Fail closed; report immediately. |
| Button+PIN | Physical five-second button hold plus parent PIN is required for core-rule-change flow. | Core change remains blocked. |
| Boot hash | Safety-critical ROM mismatch remains in permanent `LIMP_SHUTDOWN`. | Audited USB reflash only. |
| Layer separation | Layer 1 is live ROM only; Layer 2 is offline evidence validation; Layer 3 never controls hardware live. | Reject build if boundary is crossed. |

## Build Instruction

Build the Trusted Friend mobile servant robot body from `hardware/trusted_friend/`. The base may fork TurtleBot3-style mechanical patterns and OpenBot-compatible ESP32 ideas, but the repository must keep third-party source artifacts outside this package until a separate license and audit pass occurs. Run `python hardware/trusted_friend/schematic_generator.py` in a CAD-capable environment to regenerate the STEP and BOM artifacts, then verify tests #70-77 in Gazebo before physical hardware motion.
